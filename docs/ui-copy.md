# UI文言メモ（余計な説明を増やさない）

## 原則

- 画面内の文言は「操作ラベル」と「必要なエラー」だけに寄せる
- 同じ内容を繰り返さない（案内文/ボタン説明文/キャンセル通知など）
- 補足説明は「ユーザーが迷う」「危険な操作」「復帰方法が必要」なときだけ追加する

## Launcher（起動・ランチャー）

- 初期画面の案内文は置かない（例: 「最初にプロジェクトを選択してください。」）
- ボタンの説明文は置かない（例: 「既存のLaTeXフォルダを開きます」「テンプレートから main.tex を…」）
- 「キャンセルしました」系の通知は出さない（status/Issues にも出さない）
- 失敗時のみ、短いエラーを表示する（必要なら Issues に集約する）
