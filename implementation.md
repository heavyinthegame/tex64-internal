# tex64 implementation

このドキュメントは、tex64 の「ユーザーができること」を機能ベースで漏れなく整理したものです。
コード構造ではなく、画面/操作単位の一覧として扱います。

## 1) 起動・プロジェクト選択（ランチャー）

- 起動時、まず「プロジェクト（フォルダ）」を選んで開始できる
- 既存のLaTeXフォルダを開く
- 新規プロジェクトをテンプレートから作成できる（例: 論文 / 講義ノート）
- ランチャー上でテンプレートを切り替えられる

## 2) ワークスペース（フォルダ）管理

- 開いているワークスペース名/パスをUI上で確認できる
- ワークスペース内を前提に、以降の編集・検索・ビルド等が動く

## 3) ファイル管理（Files/エクスプローラー）

- ファイルツリーでフォルダ/ファイルを閲覧できる（`.git` や `node_modules` 等は基本的に表示対象から除外）
- クリックでファイルを開ける
- 右クリックの独自メニューで操作できる
  - 新しいファイル作成 / 新しいフォルダ作成
  - 名前の変更
  - 削除（即時。内部的にはゴミ箱へ移動）
  - Finderで表示
- ドラッグ&ドロップでファイル/フォルダを移動できる
- ファイルツリーにフォーカスがある状態で、ショートカット操作できる
  - コピー/カット/ペースト（⌘C/⌘X/⌘V）
  - ファイル操作のUndo（⌘Z）
- 変更が未保存のファイルが絡む名前変更/移動は、保護のため拒否される（保存を促される）

## 4) テキスト編集（エディタ）

- Monacoエディタで `.tex` や `.bib` などのテキストを編集できる
- 編集内容は短い遅延で自動保存される（加えて ⌘S でも保存できる）
- タブで複数ファイルを開ける（タブに未保存状態の表示あり）
- タブは閉じられる（閉じる際の確認ダイアログは基本なし）
- 2ペイン（primary/secondary）の分割ビューに切り替えられる
- タブをドラッグして、ペイン間に移動できる

## 5) プレビュー（PDF/画像/非対応ファイル）

- 画像ファイルはプレビュー表示できる
- PDFファイルはプレビュー表示できる
- 非対応ファイルは「非対応」としてメッセージ表示される
- PDFはスクロール閲覧でき、⌘/Ctrl＋ホイール（またはピンチ）でズームできる
- 「PDFをタブ内で開く」か「別ウィンドウで開く」かを設定で切り替えられる

## 6) ビルド（LaTeXコンパイル）

- ビルドボタンで `latexmk` によるコンパイルを実行できる
- コンパイルエンジンを選べる（例: `lualatex` / `pdflatex` / `xelatex` / `uplatex`）
- ビルド成功時にPDFを更新して表示できる（失敗時は新しいPDFに更新しない）
- ビルドログを保存し、Issuesの「詳細ログ」で見られる

## 7) 整形（フォーマッタ）

- 整形ボタンで、現在の `.tex` を整形できる（主に `latexindent`）
- 整形ルールを設定で調整できる（インデント、begin/end、空行、`&` 揃え、verbatim系の扱い等）
- `latexindent` が未導入/失敗の場合、警告としてIssuesに出しつつフォールバック動作を行う

## 8) SyncTeX（TeX→PDFジャンプ）

- 「ビルド成功時に自動でPDFへジャンプ（forward）」をON/OFFできる
- PDF上で Ctrl/Cmd+Click した位置からソースへ逆ジャンプ（reverse）。設定でON/OFFできる
- 逆SyncTeXは±3行程度の補正を行う
- PDF表示が「タブ」でも「別ウィンドウ」でもジャンプに追従できる
- `.tex` 以外では利用できない／PDFがまだ無い場合はエラーになる

## 9) アウトライン（Outline）

- “いま開いているファイル”について、以下を一覧できる
  - 章節
  - ラベル
  - 参考文献（引用キー）
  - TODO
- クリックで該当箇所へジャンプできる（基本は右側ペインで開く）

## 10) 検索（Search）

- ワークスペース内の `.tex` を検索できる（結果はファイルごとにまとまる）
- クリックで該当行へジャンプできる（基本は右側ペインで開く）
- 最大件数に上限がある（多すぎると打ち切り）

## 11) シンボル一括リネーム（Search内）

- `\\label / \\ref / \\cite` のキーを「旧→新」でまとめて更新する提案を作れる
- 対象を「ラベル/参照」「引用キー（.bib含む）」で選べる
- 実処理は“AI提案”として出し、差分を見てから適用できる

## 12) Issues（エラー/警告の集約）

- ビルド/整形/保存などのエラーや警告を一覧で確認できる
- クリックで該当行へジャンプできる
- 実行環境不足（latexmk等）系のエラーは、クリックで「実行環境」設定ページへ誘導される
- ビルドログの詳細表示（折りたたみ）を持つ

## 13) ブロック（数式編集支援）

- 数式の“挿入”ができる
  - 入力 → 差分プレビュー → 確定、の流れで挿入/置換する（自動確定しない）
  - 挿入形式を選べる（インライン/別行/`align*`/`gather*`/囲まない など）
  - インラインや別行の囲み方（`$...$` と `\\(\\)`、`$$...$$` と `\\[\\]`）を切り替えられる
- 数式の“編集”ができる
  - カーソル位置周辺の数式を自動検出してハイライトし、編集モードで差し替えられる（verbatim系は除外）
- 数式キーボード（集合/論理/矢印など）で記号を挿入できる
- “数式取り込み”で、画面キャプチャ→数式OCR→LaTeX化→入力欄へ反映できる（環境/権限が必要）

## 14) AI（サイドバーAI）

- 複数チャットを作って切り替えられる（チャット一覧⇄チャット画面）
- メッセージ送信→AI応答（進捗表示あり）
- モード操作ができる
  - 論文モード（論文向けに寄せる）
  - 自律（提案→適用→次の提案…のループ）
- 次へ（追加提案要求）
- 停止（中断）
- AIは“変更案（提案）”として出し、ユーザーが差分を見てから適用できる（ファイル編集・リネーム・作成・削除など）

## 15) UIカスタマイズ（操作性）

- サイドバーはリサイズできる（ドラッグで幅変更）
- サイドバーの主要タブは右クリックで表示/非表示を切り替えられる（最低1つは残る）
- 変更適用はすべて「差分プレビュー→確定/キャンセル」で統一されている（ブロック/AI）

## 主な制限・前提（ユーザー体験に影響する点）

- TeX環境（`latexmk`/各エンジン/`latexindent`/`synctex`）が必要。設定の「実行環境」からチェックとインストール導線がある
- 画面キャプチャはOS側の画面収録許可が必要
- ブロック挿入は `.tex` 以外ではできない

## 参考: VS Code（LaTeX Workshop）と比べて tex64 にまだ無い/弱い可能性が高い機能

 - SyncTeX の詳細設定/体験（修飾キーのカスタム、ジャンプ先の選択、候補が複数ある場合のUI、任意タイミングの手動 forward など）
- 自動ビルド（保存/依存ファイル変更で自動コンパイル）＋監視/遅延の制御
- Lint（ChkTeX/LaCheck 等）によるコンパイル前の指摘（onType/onSave など）
- Intellisense 強化（コマンド/環境/パッケージ由来補完、`\\input`/`\\include`/`\\includegraphics` のパス補完、画像プレビューなど）
- Hover プレビュー（数式レンダ、画像、cite/ref 詳細、texdoc 連携など）
- プロジェクト全体の Structure/Outline（include 先も含めた構造ツリー、カーソル追従など）
- Word count（texcount）表示
- 補助ファイル掃除（clean）/自動 clean
- `.bib` 支援の強化（`@` でのエントリ雛形、フィールド補完、整形/ソート/align など）

※ VS Code 勢が最初に“不便”と感じやすい優先度候補: 逆 SyncTeX / 自動ビルド / Lint / 補完・hover
